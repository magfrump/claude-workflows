# Plan: ui-visual-review epistemic transparency check

## Goal
Add a check to `skills/ui-visual-review.md` that flags AI-generated content
rendered alongside human-authored content when the AI section lacks a
persistent legible signal (icon, frame, or label). Hover tooltips and fine
print do not satisfy the check.

## Research notes
- Section 9 ("Accessibility serialization") is the existing
  accessibility/disclosure subsection — the closest fit per task instructions.
  It already covers ARIA labels, alt text, semantic structure, focus order,
  and "color must never be the sole signal." Epistemic transparency is the
  same family of concerns (perceivability of content metadata) and slots in
  naturally there.
- Sections 8, 9, and 10 follow an "Activation trigger → mechanical pattern
  check → common failure modes" structure. The new check should match.
- The task is explicit that the skill should *flag absence of label* on
  content the diff itself marks as AI — not auto-classify arbitrary content.
  This keeps the trigger mechanical and bounded: only fires when the diff
  contains known LLM-output markers (component names, props, SDK imports).

## Design decisions
1. **Placement.** Append to section 9 (Accessibility serialization) as a
   labelled sub-paragraph beginning **Epistemic transparency for
   AI-generated content.** Task instructed "under the existing
   accessibility/disclosure subsection," which maps to section 9. Avoids
   adding an 11th top-level item or renumbering.
2. **Trigger scope.** Activation requires explicit LLM markers in the diff:
   - Component or element names: `AIResponse`, `AIMessage`,
     `AssistantMessage`, `GeneratedContent`, `LLMOutput`, `ModelResponse`,
     `Completion`, `AIGenerated`
   - Props / template vars / state: `aiGenerated`, `isAI`, `fromAI`,
     `aiContent`, `llmResponse`, `generated`, `modelResponse`,
     `role === "assistant"`
   - LLM SDK imports adjacent to render code: `@ai-sdk/*`,
     `@anthropic-ai/sdk`, `openai`, `useChat`, `useCompletion`
3. **What counts as a persistent signal.** Visible text label
   ("AI-generated", "Generated by AI"), or an icon paired with an
   accessible name (`aria-label` / visible caption), or a labelled
   frame/badge. Explicitly disqualified: hover-only tooltips, `title`
   attributes, fine print, color-only differentiation.
4. **Severity calibration.** When AI content renders adjacent to
   human-authored content, missing label is Critical (the user cannot
   distinguish the two). When the diff renders only AI content (chat-only
   surface), missing label is Major — still recommended but lower stakes.
5. **No auto-classification.** Restate explicitly: the check flags absence
   of a label on content the diff *itself* tags as AI. The skill does not
   read prose and decide it "looks AI-generated." That keeps the check
   mechanical and avoids the obvious failure mode of false-positiving on
   human content that happens to read like an LLM.

## Implementation steps
1. Append a structured paragraph block to the end of section 9 in
   `skills/ui-visual-review.md` with: lead sentence (the requirement),
   activation trigger (LLM markers), scope (flag absence, not
   classification), severity note.
2. Commit with conventional-commit prefix
   `feat(ui-visual-review): add epistemic transparency check for AI-generated content`.
3. Push.
