# Plan: Completion Signals Guide

## Scope
Create a standalone quick-reference checklist of yes/no completion signals for RPI, DD, spike, and pr-prep workflows, plus cross-references from each workflow.

## Approach
Extract the implicit "done" criteria from each workflow phase and reframe them as self-check questions. Keep the guide short and scannable. Add one cross-reference line per workflow.

## Steps

1. Create `guides/completion-signals.md` with 2-3 yes/no questions per major phase for each workflow (~80 lines)
2. Add cross-reference line to RPI's "When to use" section
3. Add cross-reference line to DD's "When to use" section
4. Add cross-reference line to spike's "When to use" section
5. Add cross-reference line to pr-prep's "When to use" section
6. Write summary to `docs/working/summary-completion-signals-guide.md`

## Size estimate
- New file: ~80-100 lines
- Each workflow edit: 1 line added

## Risks
- None significant — purely additive documentation
